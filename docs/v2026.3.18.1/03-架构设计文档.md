## 1. 架构概述

### 1.1 架构目标
- **模块化**: 各功能模块职责清晰，独立演进
- **可测试**: 核心业务逻辑可单元测试
- **可扩展**: 支持新识别规则、新打码样式、新导出方式
- **高可用**: 外部服务失败时优雅降级

### 1.2 架构风格
- **架构模式**: 分层架构 + 管道模式
- **UI框架**: 桌面GUI (PyQt6)
- **并发模型**: 主线程UI + 工作线程处理

## 3. 模块设计

### 3.1 模块划分

| 模块 | 文件 | 职责 |
|------|------|------|
| 入口 | `__main__.py` | 程序入口，初始化QApplication |
| 主窗口 | `main_window.py` | 主窗口UI组装、业务协调 |
| UI组件 | `ui.py` | 自定义控件、工作线程、截图选择器 |
| 业务核心 | `core.py` | 识别算法、打码算法、API调用 |
| 配置 | `config.py` | 全局配置、正则规则、样式表 |
| 工具 | `utils.py` | 日志、敏感信息脱敏等工具函数 |

### 3.2 核心类设计

#### 3.2.1 MainWindow (主窗口控制器)

```python
class MainWindow(QMainWindow):
    """
    主窗口控制器，负责UI组装和业务协调
    
    职责:
    - 构建工具栏、画布区域、侧边栏
    - 处理用户操作（截图、打开、保存等）
    - 协调工作线程与UI更新
    - 管理全局快捷键
    """
    
    # 信号
    _screenshot_signal = pyqtSignal()
    
    # 核心方法
    _setup_ui()           # 初始化UI
    _make_toolbar()       # 构建工具栏
    _make_right_panel()   # 构建侧边栏
    _process_image()      # 启动图片处理流程
    _on_process_finished() # 处理完成回调
    _on_process_error()   # 处理错误回调
```

#### 3.2.2 AnnotationCanvas (打码画布)

```python
class AnnotationCanvas(QLabel):
    """
    自定义打码画布，支持显示、交互、打码预览
    
    职责:
    - 显示图片和打码效果
    - 处理鼠标事件（拖拽绘制、点击删除）
    - 管理打码区域列表
    - 应用打码并输出结果
    """
    
    # 信号
    regions_changed = pyqtSignal()
    
    # 核心方法
    load_image()          # 加载图片
    _fit_and_render()     # 自适应缩放并渲染
    _render()             # 渲染画布
    mousePressEvent()     # 鼠标按下
    mouseMoveEvent()      # 鼠标移动
    mouseReleaseEvent()   # 鼠标释放
    _find_region_at()     # 查找点击位置的区域
    undo_last()           # 撤销上一个
    clear_all()           # 清空所有
    get_result_image()    # 获取结果图片
```

#### 3.2.3 ProcessWorker (处理工作线程)

```python
class ProcessWorker(QThread):
    """
    后台处理工作线程，避免阻塞UI
    
    职责:
    - 执行云端视觉识别
    - 执行规则匹配
    - 执行AI语义分析
    - 报告进度和结果
    """
    
    # 信号
    signals.progress        # 进度消息
    signals.progress_value  # 进度值(0-100)
    signals.finished        # 处理完成
    signals.error           # 错误
    
    # 处理流程
    run()
    ├── detect_by_cloud_vision()   # OCR识别
    ├── detect_by_rules()          # 规则匹配
    └── detect_by_ai()             # AI分析
```

### 4.2 手动打码流程

```
用户操作                        系统响应
──────────────────────────────────────────────────
点击添加模式
        │
        ▼
  设置 mode="add"
  设置 cursor=CrossCursor
        │
        ▼
鼠标按下 (MousePress)
        │
        ▼
  记录 drag_start = 当前坐标
        │
        ▼
鼠标移动 (MouseMove)
        │
        ▼
  更新 drag_current = 当前坐标
  调用 _render() 显示虚线框
        │
        ▼
鼠标释放 (MouseRelease)
        │
        ▼
  计算实际坐标（减去偏移）
  转换为原图坐标（除以缩放）
  添加到 regions 列表
  调用 _render() 应用打码
  发射 regions_changed 信号
```

### 4.3 坐标转换流程

```
鼠标坐标 (屏幕坐标系)
        │
        ▼ 减去 image_offset (x, y)
相对于图片的显示坐标
        │
        ▼ 除以 display_scale
原图坐标 (用于存储bbox)
        │
        ▼ 乘以 display_scale
显示坐标 (用于渲染)
        │
        ▼ 加上 image_offset (x, y)
绘制坐标 (屏幕坐标系)
```

### 5.3 架构决策记录

#### ADR-001: 使用 PyQt6 而非 PyQt5
**状态**: 已采纳 (v2026.3.18.1)

**背景**: PyQt5 在 Windows 高 DPI 屏幕上有兼容性问题。

**决策**: 迁移到 PyQt6。

**后果**:
- ✅ 更好的高 DPI 支持
- ✅ 更现代的 API 设计
- ❌ 需要修改部分枚举常量

#### ADR-002: 使用工作线程处理图片
**状态**: 已采纳

**背景**: 图片处理和 API 调用耗时较长，阻塞 UI 会导致卡顿。

**决策**: 使用 QThread 在后台处理，通过信号与主线程通信。

**后果**:
- ✅ UI 保持响应
- ✅ 可显示进度条
- ❌ 需要处理线程同步

#### ADR-003: 云端识别而非本地 OCR
**状态**: 已采纳

**背景**: 本地 OCR 模型体积大、识别精度有限。

**决策**: 使用云端多模态大模型进行 OCR。

**后果**:
- ✅ 识别精度高
- ✅ 无需本地模型文件
- ❌ 依赖网络连接
- ❌ 有 API 调用成本

## 6. 数据流

### 6.1 图片数据流

```
PIL.Image (截图/打开)
        │
        ▼ pil_to_cv()
np.ndarray (OpenCV BGR)
        │
        ▼ cv2.imencode()
bytes (JPEG)
        │
        ▼ base64.b64encode()
str (Base64)
        │
        ▼ HTTP POST
JSON (API请求)
        │
        ▼ API响应
JSON (OCR结果)
        │
        ▼ 坐标转换
List[Region] (打码区域)
        │
        ▼ apply_mosaic()
np.ndarray (打码后图片)
        │
        ▼ cv2.imwrite()
File (保存)
        │
        ▼ QImage
Clipboard (复制)
```
