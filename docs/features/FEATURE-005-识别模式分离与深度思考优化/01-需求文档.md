## 1. 原始需求

将识别模式分离为两个独立维度，并修复VLM API兼容性错误。

### 1.1 需求背景
- 原有的"快速模式/深度模式"混合了VLM视觉识别和LLM语义识别两个不同概念
- 用户难以理解各模式的具体含义和速度差异
- VLM API的enable_thinking参数不是所有模型都支持，导致400错误

### 1.2 需求目标
1. 将识别模式分离为两个独立维度：
   - 视觉识别模式（控制VLM是否启用深度思考）
   - 敏感信息识别（控制使用规则匹配还是LLM语义分析）
2. 修复VLM API 400错误（enable_thinking参数兼容性）

---

## 2. 实现内容

### 2.1 界面重构

**新的界面结构**（从上到下）：

```
1. 打码样式
2. 模糊强度
3. 视觉识别模式  ← 新增独立章节
   - 下拉框：🚀 快速 / 🔍 深度
   
4. 敏感信息识别  ← 新增独立章节
   - ☑ 启用规则匹配（默认勾选）
   - ☐ 启用LLM语义识别（默认不勾选）
   
5. VLM & LLM 模型
6. 处理进度
7. 识别统计
```

### 2.2 功能逻辑

| 模式组合 | VLM enable_thinking | 规则匹配 | LLM语义 | 速度 | 适用场景 |
|---------|---------------------|---------|--------|------|---------|
| 快速+规则 | ❌ | ✅ | ❌ | 最快 | 格式清晰的敏感信息 |
| 快速+LLM | ❌ | ✅ | ✅ | 中等 | 需要语义理解 |
| 深度+规则 | ✅ | ✅ | ❌ | 中等 | VLM深度识别 |
| 深度+LLM | ✅ | ✅ | ✅ | 最慢 | 最高准确度 |

**默认值**：快速 + 规则（速度最快）

### 2.3 API兼容性修复

- 添加 `VLM_MODELS_SUPPORT_THINKING` 模型支持列表
- `enable_thinking` 参数仅在模型支持时才传递
- 不支持的模型使用默认参数，避免400错误

**支持的模型列表**：
- Pro/zai-org/GLM-5
- Pro/zai-org/GLM-4.7
- Qwen/Qwen3-8B/14B/32B/30B-A3B
- Qwen/Qwen3.5-397B-A17B/122B-A10B/35B-A3B/27B/9B/4B
- deepseek-ai/DeepSeek-V3.2/V3.1-Terminus
- tencent/Hunyuan-A13B-Instruct
- zai-org/GLM-4.6/GLM-4.5V
