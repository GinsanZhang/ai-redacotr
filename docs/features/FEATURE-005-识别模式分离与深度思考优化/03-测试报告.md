## 3. Bug修复记录

### Bug-001: VLM API 400错误

**严重级别**: 高
**状态**: 已修复

**问题描述**:
```
VLM识别流式请求返回 status=400
```

**根本原因**:
`enable_thinking` 参数不是所有模型都支持，对不支持的模型传递该参数导致400错误。

**修复方案**:
1. 添加 `VLM_MODELS_SUPPORT_THINKING` 支持列表
2. 仅当模型在支持列表中且选择深度模式时才传递参数
3. 其他情况使用默认参数

**验证结果**: ✅ API调用正常，无400错误

---

### Bug-002: 缩进错误导致应用无法启动

**严重级别**: 高
**状态**: 已修复

**问题描述**:

**根本原因**:
代码修改时缩进不一致

**修复方案**:
统一缩进格式，确保所有代码块使用正确的缩进级别
