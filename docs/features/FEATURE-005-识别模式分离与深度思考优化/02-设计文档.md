## 1. UI设计

### 1.1 界面布局变更

**原有布局**:
```
打码样式
模糊强度
识别模式 ← 单一下拉框（快速/深度）
VLM & LLM 模型
处理进度
识别统计
```

**新布局**:
```
打码样式
模糊强度
视觉识别模式 ← 新增下拉框
   🚀 快速
   🔍 深度
敏感信息识别 ← 新增复选框
   ☑ 启用规则匹配
   ☐ 启用LLM语义识别
VLM & LLM 模型
处理进度
识别统计
```

### 1.2 控件设计

#### 视觉识别模式（ComboBox）
- 类型：QComboBox
- 选项：
  - "🚀 快速" (data: "fast")
  - "🔍 深度" (data: "deep")
- 默认：快速
- Tooltip：说明快速/深度的区别

#### 敏感信息识别（CheckBox）
- 启用规则匹配
  - 默认：勾选
  - Tooltip：基于正则表达式识别格式清晰的敏感信息
- 启用LLM语义识别
  - 默认：不勾选
  - Tooltip：使用LLM进行语义分析识别敏感信息

## 2. 架构设计

### 2.1 配置结构

```python
CONFIG = {
    # VLM配置
    "vlm_api_url": "...",
    "vlm_api_key": "...",
    "vlm_model": "Qwen/Qwen3-VL-32B-Instruct",
    "vlm_mode": "fast",  # 新增：fast|deep
    
    # LLM配置
    "llm_api_url": "...",
    "llm_api_key": "...",
    "llm_model": "...",
    "ai_enabled": False,  # 是否启用LLM语义识别
}
```

### 2.2 模块关系

```
MainWindow
    ├── vlm_mode_combo (ComboBox)
    │   └── 信号 → _on_vlm_mode_changed() → CONFIG["vlm_mode"]
    │
    ├── rule_checkbox (CheckBox)
    │   └── 始终启用（预留扩展）
    │
    └── llm_checkbox (CheckBox)
        └── 信号 → CONFIG["ai_enabled"]
            ↓
        ProcessWorker.run()
            ↓
        detect_by_vlm(vlm_mode=CONFIG["vlm_mode"])
            ↓
        detect_by_ai() ← 仅当 ai_enabled=True
```

### 2.3 API调用逻辑

```python
def detect_by_vlm(image, vlm_mode="fast"):
    body = {
        "model": vlm_model,
        "messages": [...],
    }
    
    # 仅当深度模式且模型支持时传递
    if vlm_mode == "deep" and model in VLM_MODELS_SUPPORT_THINKING:
        body["enable_thinking"] = True
    
    # 调用API...
```

## 4. 兼容性说明

### 4.1 当前默认模型

- 默认VLM模型：`Qwen/Qwen3-VL-32B-Instruct`
- 该模型**不在**支持列表中
- 因此启用深度模式也不会传递 `enable_thinking`
- 这是预期行为，不影响功能使用

### 4.2 向后兼容

- 原有的 `ai_enabled` 配置项继续使用
- 新增的 `vlm_mode` 配置项默认为 "fast"
- 旧代码逻辑保持兼容
