## 1. 功能概述

### 1.1 需求背景
原有的"快速模式/深度模式"混合了两个不同概念，用户难以理解。同时VLM API的enable_thinking参数不是所有模型都支持，导致400错误。

### 1.2 实现功能

✅ **核心功能**:
1. 视觉识别模式独立控制（快速/深度）
2. 敏感信息识别独立控制（规则/LLM）
3. VLM API兼容性修复

## 3. 功能演示

### 3.1 四种模式组合

| 组合 | 说明 | 速度 |
|------|------|------|
| 快速+规则 | 最快速识别，适合格式清晰的敏感信息 | 最快 |

| 深度+规则 | VLM深度思考+规则匹配 | 中等 |
| 深度+LLM | 最高准确度，较慢 | 最慢 |

## 4. Bug修复记录

### Bug-001: VLM API 400错误（主要修复）

**问题**: enable_thinking参数对不支持模型传递导致400错误

**修复**:
- 添加VLM_MODELS_SUPPORT_THINKING支持列表
- 仅对支持的模型传递参数

**验证**: ✅ API调用正常

### Bug-002: 缩进错误

**问题**: 代码缩进不一致导致应用无法启动

## 6. 已知限制

1. **当前默认模型不支持enable_thinking**

   - 选择深度模式不会传递enable_thinking参数
   - 建议更换为支持的模型以使用深度思考功能

2. **规则匹配复选框**
