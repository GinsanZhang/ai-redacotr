# 需求文档
## FEATURE-001: 识别流程手动触发

---

## 1. 原始需求描述

**提出日期**: 2026-03-18
**提出人**: 产品经理

**需求描述**:  
现在是自动进行AI识别，用户希望是有个按钮手动触发。

---

## 2. 需求分析

### 2.1 当前行为
- 截图后自动触发云端视觉识别(OCR)
- 自动触发规则匹配
- 自动触发AI语义识别（如果开启）
- 用户无法控制识别流程

### 2.2 期望行为
- 截图后**不进行任何自动识别**
- **用户手动点击"开始识别"按钮后，才进行完整识别流程**
- 识别流程包括：OCR识别 → 规则匹配 → AI语义识别
- 给用户完全控制权，减少不必要的API调用

### 2.3 技术方案
1. 在侧边栏添加"开始识别"按钮（或工具栏）
2. 截图后仅加载图片，不启动识别流程
3. 用户点击按钮后启动ProcessWorker进行完整识别
4. 按钮状态管理：识别前启用，识别中禁用，识别完成后启用（允许重新识别）

---

## 3. 功能范围

### 3.1 包含的功能
- ✅ 添加"开始识别"按钮（工具栏或侧边栏）
- ✅ 截图后不再自动启动识别流程
- ✅ 用户点击按钮后启动完整识别流程（OCR+规则+AI）
- ✅ 按钮状态管理：识别前启用，识别中禁用，识别完成启用
- ✅ 进度条和状态栏正常更新
- ✅ 支持重新识别（识别完成后可再次点击）

### 3.2 不包含的功能
- ❌ 分阶段识别（不复用原流程的阶段性暂停）
- ❌ 修改截图流程
- ❌ 修改打码样式和导出功能
- ❌ 修改快捷键

---

## 4. 验收标准

### 4.1 功能验收
| 验收项 | 验收标准 | 优先级 |
|--------|----------|--------|
| 按钮显示 | 工具栏或侧边栏显示"开始识别"按钮，样式与其他按钮一致 | P0 |
| 不自动识别 | 截图后图片加载但不自动启动识别，按钮状态为可用 | P0 |
| 手动触发 | 点击按钮后启动完整识别流程（OCR+规则+AI） | P0 |
| 状态提示 | 截图后状态栏显示"点击开始识别"或类似提示 | P0 |
| 进度更新 | 识别过程中进度条正常更新（5%->55%->75%->100%） | P1 |
| AI开关 | 复选框"启用AI识别"仍有效，控制是否执行AI识别阶段 | P1 |
| 重新识别 | 识别完成后可再次点击按钮重新识别 | P2 |

### 4.2 性能验收
- OCR完成到按钮可用延迟 < 100ms
- 点击按钮到AI识别开始延迟 < 100ms

---

## 5. UI设计

### 5.1 按钮位置
- 位置：建议放在工具栏（与其他操作按钮一起）
- 按钮文字: "🔍 开始识别"
- 按钮ID: btnStartRecognition
- 样式：使用btnPrimary（蓝色主按钮）
- 初始状态：禁用（未加载图片时）

### 5.2 状态流转
```
截图/打开图片后
  ↓
[图片已加载] 按钮变为可用
  ↓ 用户点击"开始识别"
[识别中...] 按钮禁用，进度条0-100%
  ├─ 5%: 云端视觉识别中...
  ├─ 55%: 规则匹配中...
  ├─ 75%: AI分析中...（如果启用）
  └─ 100%: 处理完成
  ↓
[识别完成] 按钮变为可用（支持重新识别）
```

---

## 6. 接口变更

### 6.1 ProcessWorker调整
- 不再在截图后自动启动
- 由按钮点击触发启动

### 6.2 MainWindow处理
修改方法：
- `_process_image()`: 不再自动启动ProcessWorker，仅加载图片并启用按钮

新增方法：
- `_start_recognition()`: 手动触发识别流程，启动ProcessWorker
