## 1. 原始需求描述

VLM识别太慢，尝试先识别独立的几大块，然后并行识别。

---

## 2. 需求分析

### 2.1 需求澄清

| 澄清项 | 结论 |
|--------|------|
| Step1 实现方式 | 轻量VLM快速全景识别（不是OCR，OCR不能返回坐标） |
| 分块策略 | 按行/列智能分块，每区域 ≤20 块 |
| 并发数 | 最大20并发 |
| 跨区域处理 | 尽量不跨区域 |

### 2.2 技术方案

```
┌─────────────────────────────────────────┐
│  Step 1: 轻量 VLM 快速全景识别            │
│  - 模型: Qwen3-VL-8B-Instruct            │
│  - 图像: 缩放到 1024                     │
│  - Prompt: 精简，只需返回文字块位置和内容  │
│  - 输出: [{"bbox": [x1,y1,x2,y2], "text": "..."}]
└────────────────────┬────────────────────┘
                     ↓
┌─────────────────────────────────────────┐
│  Step 2: 智能分块（不跨区域）              │
│  - 按行分组（y坐标相近的为一行）           │
│  - 每组 ≤ 20 块（并发上限）               │
│  - 计算每组区域的边界框                    │
└────────────────────┬────────────────────┘
                     ↓
         ┌───────────┼───────────┐
         ↓           ↓           ↓
    ┌─────────┐ ┌─────────┐ ┌─────────┐
    │ 区域 1  │ │ 区域 2  │ │ 区域 3  │  ← 并行调用VLM
    │ 裁剪    │ │ 裁剪    │ │ 裁剪    │
    │ 详细识别│ │ 详细识别│ │ 详细识别│
    └────┬────┘ └────┬────┘ └────┬────┘
         └───────────┼───────────┘
                     ↓
┌─────────────────────────────────────────┐
│  Step 3: 合并结果，按原始位置排序         │
└─────────────────────────────────────────┘
```

### 2.3 模型配置

在 `config/models.json` 中添加轻量模型：

```json
{
  "vlm_models": {
    "lightweight": "Qwen/Qwen3-VL-8B-Instruct"
  }
}
```

## 3. 验收标准

| 编号 | 验收项 | 验收方式 |
|------|--------|----------|
| AC-01 | 整体识别速度比单次VLM调用快 | 性能测试 |
| AC-02 | 识别结果准确性不降低 | 对比测试 |
| AC-03 | 支持并发控制（max_workers=20） | 代码检查 |
| AC-04 | 分块不破坏文字完整性 | 功能测试 |
| AC-05 | 轻量模型配置可自定义 | 配置测试 |

## 5. 风险评估

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| 轻量模型识别不完整 | 中 | 高 | 回退到单次完整识别 |
| 并发数过高触发限流 | 低 | 中 | 限制 max_workers=20 |
| 分块边界切割文字 | 中 | 中 | 按行分组，避免切割 |
