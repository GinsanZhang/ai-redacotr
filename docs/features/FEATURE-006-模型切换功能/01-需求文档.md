## 1. 原始需求描述

实现一个功能，界面上可切换模型列表，配置的时候支持在文件配置好模型清单。

---

## 2. 需求分析

### 2.1 需求澄清

| 澄清项 | 结论 |
|--------|------|
| 功能范围 | VLM视觉模型 + LLM敏感信息识别模型 |
| 配置格式 | JSON格式配置文件 |
| UI方式 | 各章节独立下拉框 |
| 用户选择记忆 | 是，记住用户选择 |

### 2.2 功能范围

**包含**:
- JSON配置文件定义VLM和LLM模型清单
- VLM识别章节添加模型下拉框
- 敏感信息识别章节添加模型下拉框
- 用户选择的模型持久化存储
- 重启后自动恢复上次选择的模型

**不包含**:
- API配置（URL、Key仍通过环境变量设置）
- 模型参数配置（timeout、retries等）
- 多API服务商切换

### 2.3 技术方案

#### 配置文件设计

新建 `config/models.json`:

```json
{
  "vlm_models": {
    "default": "Qwen/Qwen3-VL-32B-Instruct",
    "models": [
      {
        "id": "Qwen/Qwen3-VL-32B-Instruct",
        "name": "Qwen3-VL 32B",
        "supports_thinking": true
      },
      {
        "id": "Qwen/Qwen2.5-VL-32B-Instruct",
        "name": "Qwen2.5-VL 32B",
        "supports_thinking": false
      }
    ]
  },
  "llm_models": {
    "default": "Pro/moonshotai/Kimi-K2.5",
    "models": [
      {
        "id": "Pro/moonshotai/Kimi-K2.5",
        "name": "Kimi K2.5"
      },
      {
        "id": "deepseek-ai/DeepSeek-V3",
        "name": "DeepSeek V3"
      }
    ]
  }
}
```

#### 用户选择持久化

新建 `config/user_settings.json` 存储用户偏好:

```json
{
  "vlm_model": "Qwen/Qwen3-VL-32B-Instruct",
  "llm_model": "Pro/moonshotai/Kimi-K2.5",
  "vlm_mode": "fast",
  "ai_enabled": false
}
```

#### UI改动

1. **视觉识别模式章节**
   - 新增模型下拉框，位于识别模式下拉框下方
   - 下拉框显示模型名称，实际使用模型ID

2. **敏感信息识别章节**
   - 新增模型下拉框，位于复选框下方
