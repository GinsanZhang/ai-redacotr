# 需求文档
## FEATURE-002: 优化AI识别流程

---

## 1. 原始需求描述

**提出日期**: 2026-03-18
**提出人**: 产品经理 (Ginsan)

**需求描述**:  
不想要第二次的AI调用，在视觉模型识别的时候，就标记对应的敏感标识。AI二次识别作为可选项，默认不开启，因为整体加上时间太长，体验不好。

---

## 2. 需求分析

### 2.1 当前流程分析

**当前识别流程**:
```
1. VLM识别 (OCR) → 获取所有文字块 [耗时: ~30-40s]
2. 规则匹配 → 识别标准格式敏感信息 [耗时: ~0.1s]
3. AI语义分析 → 识别非标准隐私信息 [耗时: ~30-40s]
```

**总耗时**: 60-80秒（用户反馈超过60秒）

**痛点**:
- AI语义分析阶段耗时较长
- 两次AI调用（OCR + AI语义）导致整体体验不佳
- 用户可能不需要AI语义识别，但无法选择

### 2.2 优化方案

**方案**: 让VLM模型（Qwen-VL）在OCR的同时识别敏感信息

**新流程**:
```
1. VLM识别 (OCR + 敏感信息识别) → 直接返回敏感区域 [耗时: ~3-5s]
2. 规则匹配 → 作为补充 [耗时: ~0.1s]
```

**优化后总耗时**: 30-40秒 (减少50%)

**AI二次识别改为可选项**:
- 默认关闭
- 用户可在设置中开启
- 开启后走原流程（OCR → 规则 → AI语义）

### 2.3 技术可行性

**Qwen-VL能力**:
- 支持VLM识别，可以在OCR时判断内容类型
- 可以通过Prompt引导识别敏感信息
- 返回格式可包含标签信息

**Prompt设计**:
```
你是一个高精度的 OCR 助手和隐私保护专家。
请识别图片中所有的文字块，并判断哪些是敏感信息（姓名、地址、手机号、身份证号等）。
重要：bbox 坐标必须是基于图片尺寸归一化到 [0, 1000] 范围内的整数。
格式：{"blocks":[{"text":"...","bbox":[xmin,ymin,xmax,ymax],"is_sensitive":true,"sensitive_type":"姓名"}]}
```

## 4. 验收标准

### 4.1 功能验收

| 验收项 | 验收标准 | 优先级 |
|--------|----------|--------|
| OCR识别敏感信息 | OCR返回结果包含敏感信息类型 | P0 |
| 识别时间优化 | 总识别时间减少50%（从60-80秒降到30-40秒） | P0 |
| AI二次识别开关 | 侧边栏有复选框，默认关闭 | P0 |
| 开关有效性 | 关闭时只进行OCR+规则，开启时进行OCR+规则+AI | P0 |
| 规则补充 | OCR未识别的标准格式（如手机号）仍被规则捕获 | P1 |
| 向后兼容 | 开启AI二次识别后，原功能正常 | P1 |

### 4.2 性能验收

- 识别耗时从60-80秒降低到30-40秒
- 准确率不降低（敏感信息识别率 ≥ 90%）

## 6. 技术方案

### 6.1 核心改动

#### 6.1.1 修改 detect_by_cloud_vision() (core.py)

**新Prompt设计**:
```python
prompt = """你是一个高精度的 OCR 助手和隐私保护专家。

请识别图片中所有的文字块，并判断哪些是敏感信息。

敏感信息类型包括：
- 姓名（人名）
- 详细地址
- 手机号
- 身份证号
- 银行卡号
- 邮箱
- 职位/职称
- 企业名称
- 其他个人隐私

重要：bbox 坐标必须是基于图片尺寸归一化到 [0, 1000] 范围内的整数。

输出格式：
{
  "blocks": [
    {
      "text": "识别文字",
      "bbox": [xmin, ymin, xmax, ymax],
      "is_sensitive": true/false,
      "sensitive_type": "姓名/地址/手机号等"
    }
  ]
}"""
```

**结果解析修改**:
```python
# 处理返回结果
for block in data["blocks"]:
    text = block["text"]
    bbox = block["bbox"]
    is_sensitive = block.get("is_sensitive", False)
    sensitive_type = block.get("sensitive_type", "")
    
    # 添加到OCR结果
    ocr_results.append({
        "text": text,
        "bbox": bbox,
        "conf": block.get("conf", 1.0)
    })
    
    # 如果是敏感信息，直接加入hits
    if is_sensitive and sensitive_type:
        hits.append({
            "bbox": bbox,
            "label": sensitive_type,
            "text": text,
            "source": "ai"  # 标记为AI识别
        })
```

#### 6.1.2 修改 ProcessWorker 流程 (ui.py)

根据配置决定流程：

```python
def run(self):
    try:
        if CONFIG.get("deep_ai_enabled", False):
            # 深度AI模式：原流程
            self._run_with_deep_ai()
        else:
            # 快速模式：OCR直接识别敏感信息
            self._run_fast_mode()
    except Exception as e:
        self.signals.error.emit(str(e))

def _run_fast_mode(self):
    """快速识别模式：OCR+规则"""
    # OCR阶段（包含敏感信息识别）
    self.signals.progress.emit("VLM识别中...")
    ocr_result = detect_by_vlm(
        self.image_cv, 
        self.signals.progress.emit,
        include_sensitive=True  # 新参数
    )
    # OCR结果中已包含敏感信息
    
    # 规则匹配作为补充
    self.signals.progress.emit("规则匹配中...")
    rule_hits = detect_by_rules(ocr_result["ocr_blocks"])
    
    # 合并结果（去重）
    all_hits = ocr_result["sensitive_hits"] + rule_hits
    
    self.signals.finished.emit(ocr_result["ocr_blocks"], all_hits, timings)
```
