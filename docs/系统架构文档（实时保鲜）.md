### 3.2 核心类设计

#### MainWindow (主窗口控制器)
```python
class MainWindow(QMainWindow):
    """主窗口控制器，负责UI组装和业务协调"""
    - _setup_ui(): 初始化UI
    - _make_toolbar(): 构建工具栏
    - _make_right_panel(): 构建侧边栏
    - _process_image(): 加载图片（不再自动识别）
    - _start_recognition(): 手动触发识别
    - _on_process_finished(): 识别完成回调
```

#### AnnotationCanvas (打码画布)
```python
class AnnotationCanvas(QLabel):
    """自定义打码画布"""
    - load_image(): 加载图片
    - _render(): 渲染画布
    - mousePressEvent(): 鼠标按下处理
    - mouseReleaseEvent(): 释放创建区域
    - get_result_image(): 获取结果图片
```

#### ProcessWorker (处理工作线程)
```python
class ProcessWorker(QThread):
    """后台处理工作线程"""
    - run(): 执行识别流程
        ├── detect_by_cloud_vision(): OCR识别
        ├── detect_by_rules(): 规则匹配
        └── detect_by_ai(): AI分析（可选）
```

### 4.2 外部接口

#### 云端视觉 API
- **端点**: `POST https://api.siliconflow.cn/v1/chat/completions`
- **鉴权**: Bearer Token
- **模型**: Qwen/Qwen3-VL-32B-Instruct
- **请求格式**: 图片Base64 + Prompt
- **响应格式**: JSON包含OCR结果

#### AI语义 API
- **端点**: `POST https://api.siliconflow.cn/v1/chat/completions`
- **鉴权**: Bearer Token
- **模型**: Pro/moonshotai/Kimi-K2.5
- **请求格式**: 文本列表 + Prompt
- **响应格式**: JSON包含敏感信息ID列表

## 5. 数据流转

### 5.1 识别流程数据流

```
PIL.Image (截图/打开)
        ↓
np.ndarray (OpenCV BGR格式)
        ↓
用户点击"开始识别"按钮
        ↓
ProcessWorker启动
        ↓
┌─────────────────────────────────────┐
│ 1. OCR识别                          │
│    - 图片压缩                       │
│    - Base64编码                     │
│    - 调用云端API                    │
│    - 解析OCR结果                    │
├─────────────────────────────────────┤
│ 2. 规则匹配                         │
│    - 遍历OCR结果                    │
│    - 正则匹配敏感信息               │
├─────────────────────────────────────┤
│ 3. AI语义识别（可选）               │
│    - 过滤已匹配项                   │
│    - LLM分析                        │
│    - 解析结果                       │
└─────────────────────────────────────┘
        ↓
List[Region] (打码区域列表)
        ↓
渲染到画布
        ↓
用户可手动调整区域
        ↓
导出结果
```

## 6. 关键设计决策

### 6.1 架构决策记录

#### ADR-001: 使用 PyQt6 而非 PyQt5
- **状态**: 已采纳 (v2026.3.18.1)
- **决策**: 迁移到 PyQt6
- **原因**: 更好的高 DPI 支持，更现代的 API
- **影响**: 需要修改部分枚举常量

#### ADR-002: 云端识别而非本地 OCR
- **状态**: 已采纳
- **决策**: 使用云端多模态大模型进行 OCR
- **原因**: 识别精度高，无需本地模型文件
- **影响**: 依赖网络连接，有 API 调用成本

#### ADR-003: 手动触发识别 (v2026.3.18.1)
- **状态**: 已采纳
- **决策**: 截图后不自动识别，等待用户手动触发
- **原因**: 给用户控制权，减少不必要的API调用
- **影响**: 修改了 _process_image() 流程
