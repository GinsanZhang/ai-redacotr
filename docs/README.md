## 🚀 快速开始

### 1. 了解项目

如果你是新成员，建议按以下顺序阅读：

1. [../README.md](../README.md) - 项目总体介绍
2. [v2026.3.18.1/01-产品需求文档.md](./v2026.3.18.1/01-产品需求文档.md) - 了解产品功能
3. [v2026.3.18.1/03-架构设计文档.md](./v2026.3.18.1/03-架构设计文档.md) - 了解技术架构
4. [系统架构文档（实时保鲜）.md](./系统架构文档（实时保鲜）.md) - 了解当前代码状态

### 2. 开发工作流

作为开发者，请遵循 [研发流程规范.md](./研发流程规范.md) 中定义的流程：

```
需求提出 → 需求确认[CP1] → 需求分析 → 设计阶段[CP2] → 
开发阶段[CP3] → 测试阶段[CP4] → 验收通过[CP5] → 已关闭
```

### 3. 查找文档

- **要开发新功能？** → 查看 [研发流程规范](./研发流程规范.md) 中的文档模板
- **要修改代码？** → 先读 [系统架构（实时保鲜）](./系统架构文档（实时保鲜）.md)
- **要了解历史版本？** → 查看 [v2026.3.18.1/](./v2026.3.18.1/) 目录
- **要跟踪需求？** → 查看 [requirements-pool.md](./requirements-pool.md)（待创建）
